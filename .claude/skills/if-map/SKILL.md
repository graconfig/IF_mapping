# if-map — 为空白设计书生成 SAP 字段候选

## 何时使用此 skill

- 用户提供一份空白（只填了连携元/源字段）的接口设计书（.xls 或 .xlsx），要求生成 SAP 侧推荐。
- 该项目已通过 `if-ingest` 建好了知识库 —— 通过 `projects/<project>/knowledge/ifs.db` 存在来确认。

## 用户视角

用户**不需要看/编 schema.yaml**。体验是：「给文件 → 稍等 → 拿到回填 Excel 与统计」。格式识别由 skill 和 AI 在后台完成；只有 AI 判断存在风险时才用自然语言问简单问题（例如"这份表的字段名列是第 C 列的'項目名'吧？"）。

## 产物

- `<input_stem>_候选.xlsx` —— 与输入同目录（通常 `projects/<project>/fill_out/`）
  - 追加 3 列：`推奨 SAP 字段`（带 Top-3 下拉）、`信心`（★ 分级）、`備考（业务判断）`
  - 业务化说明文字；复合字段自动拆分；无历史映射的字段也给"推测"候选（`[推测]` 前缀）

## 执行步骤

### 1. 接收输入
- **项目名**：从上下文或目录推断（如文件在 `projects/kaps2/fill_out/` → `kaps2`）。不确定时用自然语言问。
- **空白设计书路径**（.xls / .xlsx）。
- 确认 `projects/<project>/knowledge/ifs.db` 存在，否则告诉用户先摄取参考文档（触发 if-ingest）。

### 2. 自动探测（后台完成，不向用户展示 yaml）
```bash
python3 scripts/detect_schema.py --as blank <input.xls> \
  --if-name "<业务名>" \
  --out <input_dir>/<input_stem>.schema.yaml
```
生成的 schema.yaml **不向用户展示**，只作为内部缓存。

### 3. AI 语义校验（关键）
用 Read 工具直接打开输入文件（若是 .xls 先用 soffice 转 xlsx 放到 /tmp），检查探测结果是否合理：
- **表头行** 是否含字段语义词（項目名 / コード / Type / 长度）
- **字段名列** 内容是否真像字段名而不是序号或技术名
- **技术名列** 是否真是技术命名（大写字母缩写 / COBOL 风格）
- **类型列** 是否真是类型代码（C/N 等单字符或 CHAR/NUMC）

若 Read 无法处理当前文件格式，用 `python3 -c "import openpyxl; ..."` 短脚本看头 15 行。

**校验通过** → 步骤 5。
**校验可疑** → 步骤 4。

### 4. 仅在需要时求助（A3 兜底）
用**业务语言**问 1-2 个具体问题（不说 "schema / header_row / columns"）：
- "这份表的字段列表从第几行开始？"
- "字段名和字段代码分别在哪一列？比如 '項目名' 列 / '項目コード' 列是第几个？"
- "有没有需要跳过的填充字段（比如 '予備' 之类）？"

根据回答**直接修改** schema.yaml（AI 代编辑），然后重跑步骤 3 验证。

### 5. 运行回填
```bash
python3 scripts/fill_book.py --project <project> <input.xls>
```
- 支持 .xls 自动转 .xlsx（soffice headless）
- 知识库查询 + 三层推测 + 复合拆分，生成 `<input_stem>_候选.xlsx`

### 6. 向用户汇报（业务语言）
从命令输出抓关键数字，用几句话告诉用户：
- "共识别出 N 个字段"
- "高信心命中 X 个、需人工复核 Y 个、推测 Z 个、接口内部字段跳过 W 个"
- 产物路径

### 7. 结束语
告诉用户：在 Excel 里打开产物，`推奨 SAP 字段`列可点击下拉切换 Top-3 或手动填写。★★★ 可直接采纳；★（推测）和"需确认"需人工复核。

## 信心度对照（向用户解释时用业务语言）

| 标签 | 建议 |
|---|---|
| ★★★ | 非常可靠，直接采纳 |
| ★★（历史仅1次，需复核）| 历史样本少，复核一下 |
| ★★ | 有主导候选 + 备选，复核后采纳 |
| ★ | 多候选并列，结合业务判断 |
| 需确认 | 几个候选频次相近，请业务方定夺 |
| ★（推测）| 无历史映射，推测而来，必须复核 |
| — | 无候选或跳过字段（接口内部字段）|
| 复合字段 | 已自动拆成两个子字段 |

## 关键注意事项

- 全流程**不主动向用户展示 schema.yaml**；AI 改这个文件也不宣布文件名。
- 原文件不修改，产物都是独立的 `_候选.xlsx`。
- 用户说"重跑" → 直接删除同名 `_候选.xlsx` 后重跑。
- 业务词典由维护者迭代：`scripts/business_dict.default.yaml`（跨项目）+ `projects/<project>/business_dict.yaml`（项目特有）—— 用户无需感知。
